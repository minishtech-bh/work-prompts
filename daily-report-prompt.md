# 일일보고 프롬프트 (CLUBMINISH)

> club-minish-backend 레포의 오늘 내 커밋을 기반으로, Slack `dx-team-2` 채널의 DX2-일일보고 워크플로 양식에 맞게 일일보고를 제출하기 위한 프롬프트입니다.

---

```
https://github.com/MinishTechnology/club-minish-backend 의 develop 브랜치에서

오늘 내가(minishtech-bh) 커밋한 내용들을 바탕으로 일일보고를 작성해서 제출해줘.

[제출 위치]

- Slack의 dx-team-2 채널에서 오늘 일자(DX2-일일보고 워크플로) 메시지의

  "일일보고 작성하기" 버튼을 눌러서 열리는 양식에 작성할 것

- 절대 업보맨 DM이나 채널에 직접 메시지로 적지 말 것

[작성 규칙]

- "오늘 한 일" 영역:

  * 최상위에 CLUBMINISH 를 글머리표(•)로 적고

  * 그 하위에 오늘 한 작업들을 들여쓴 글머리표(○)로 나열

  * 줄바꿈은 Shift+Enter, 들여쓰기는 Tab 사용 (절대 Enter 단독으로 누르지 말 것 — 양식 밖에서는 메시지가 전송됨)

  * 글머리표는 Cmd+Shift+8 로 시작

- "내일 계획" 영역도 동일 포맷 (CLUBMINISH > 하위 항목)

- "특이사항" 영역은 옵션 항목 — 아래에 내용이 있으면 그대로 적고,

  "없음" 또는 비어있으면 해당 필드는 손대지 말 것

- 커밋 메시지를 그대로 옮기지 말고, 비개발자(기획/디자인/PM)도 이해할 수 있는

  자연스러운 한국어로 의역할 것. 너무 세세하지 않게, 기능/이슈 단위로 묶어서.

- Merge 커밋은 본 작업 커밋과 중복되므로 한 항목으로 합칠 것

- 영문 기술 용어(enum, deserializer, outbox 등)는 가급적 풀어쓰되,

  꼭 필요한 경우만 괄호로 병기할 것

[내일 할 일]

프론트 API 연동 버그 해결

※ 매일 이 부분만 그날 상황에 맞게 교체해서 보낼 것

[특이사항]

없음

※ 매일 이 부분만 그날 상황에 맞게 교체해서 보낼 것
   (예: "오후 반차", "@김범창(FE)에게 API 스펙 공유 필요", "내일 외근으로 오전만 근무" 등)
   특별히 공유할 내용 없으면 "없음" 그대로 두기

[제출 전 체크]

- 양식 안에서 CLUBMINISH가 1단 글머리표, 하위 항목들이 2단 글머리표로
  들여쓰기 되어 있는지 스크린샷으로 확인

- 확인되면 "제출" 버튼 클릭, "워크플로 완료" 토스트 확인

오늘 커밋이 0건이면 보고를 제출하지 말고 먼저 나에게 알려줘.
```

# 주간보고 프롬프트 (CLUBMINISH)

> club-minish-backend 레포의 이번 주 내 커밋을 기반으로, 주간보고 양식에 붙여넣을 텍스트를 채팅창에 정리해서 받기 위한 프롬프트입니다. (자동 제출 X, 직접 복붙용)

---

```
https://github.com/MinishTechnology/club-minish-backend 의 develop 브랜치에서

이번 주(이번 주 월요일 0시 ~ 오늘까지) 내가(minishtech-bh) 커밋한 내용들을 바탕으로 주간 업무 보고를 작성해줘.

[제출 방식]

- Slack/노션 등에 자동 제출하지 말고, 채팅창에 텍스트로만 정리해서 보여줄 것

- 내가 직접 복사해서 주간보고 양식에 붙여넣을 거임

[보고 양식]

- 진행하셨던 업무

- 다음주 진행 예정 업무

- 특이 사항(있는 경우)

[작성 규칙]

- "진행하셨던 업무" 영역:

  * 이번 주 작업을 5개 항목 내외로 압축 (3~6개 사이에서 적당히 조절)

  * 글머리표(번호 또는 -) 단순 리스트로 작성,
    일일보고처럼 CLUBMINISH 들여쓰기는 사용하지 말 것

  * PR/이슈/기능 단위로 묶을 것 — 같은 PR에 속한 자잘한 리팩/문서/테스트 커밋은
    본 작업 항목에 흡수

  * Merge 커밋은 본 작업과 중복이므로 합쳐서 한 줄로

  * 자잘한 운영 개선/안정성 작업이 여러 개면 마지막 항목에
    "운영 개선 및 안정성 작업 다수 (예: ~, ~, ~)" 식으로 묶을 것

  * 커밋 메시지를 그대로 옮기지 말고, 비개발자(기획/디자인/PM)가 한 줄만 읽어도
    무엇이 좋아졌는지 이해할 수 있는 자연스러운 한국어로 의역

  * 영문 기술 용어(enum, deserializer, outbox 등)는 풀어쓰되,
    꼭 필요한 경우에만 괄호로 병기

  * "어떤 화면/어떤 사용자(플랫폼 어드민/파트너/입점사/임직원 등)에게
    무엇이 바뀌었는지"가 드러나도록 서술

- "다음주 진행 예정 업무" 영역:

  * 기본값은 "버그 수정 및 QA 대응" 한 줄

  * 그 외 미리 정해진 다음주 작업이 있으면 별도로 알려줄 테니,
    알려준 내용으로 교체할 것

- "특이 사항" 영역:

  * 기본값은 "없음" — 없으면 해당 줄은 비워두거나 "없음"으로 적기

  * 별도로 알려주는 경우(예: "월요일 오전 반차", "수요일 외근",
    "@김도현(FE)에게 API 스펙 공유 필요" 등)에만 그대로 적기

[제출 전 체크]

- 항목 수가 5개 내외(최대 6개)인지 확인

- 각 항목이 비개발자가 읽어도 무엇인지 짐작 가능한지 확인

- Merge 커밋 / 동일 PR 내 자잘한 커밋이 본 항목에 흡수돼 있는지 확인

- 출력 후 "이대로 복사해서 쓰면 됩니다" 한 줄 덧붙일 것

이번 주 커밋이 0건이면 보고를 만들지 말고 먼저 나에게 알려줘.

[다음주 예정 업무]

버그 수정 및 QA 대응

※ 매주 이 부분만 그 주 상황에 맞게 교체해서 보낼 것

[특이사항]

없음

※ 매주 이 부분만 그 주 상황에 맞게 교체해서 보낼 것
   (예: "월요일 오전 반차", "수요일 외근",
   "@김도현(FE)에게 API 스펙 공유 필요" 등)
   특별히 공유할 내용 없으면 "없음" 그대로 두기
```
